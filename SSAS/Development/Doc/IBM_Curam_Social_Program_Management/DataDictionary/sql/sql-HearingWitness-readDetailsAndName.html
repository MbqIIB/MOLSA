Database SQL Operation: HearingWitness.readDetailsAndName

Operation Details
Description: Returns the details of the hearing witness. As the hearing witness details are stored in more than one table, a standard read can't be used.
Sterotype: ns

SQL 
SELECT
  HW.hearingWitnessID,
  HW.hearingID,
  HW.voluntaryIndicator,
  HW.participatedCode,
  HW.behalfOfCode,
  HW.comments,
  HW.versionNo,
  CR.concernRolename,
  HW.caseParticipantRoleID,
  CPR.versionNo,
  CPR.recordStatus,
  CPR.participantRoleID
INTO
  :hearingWitnessID,
  :hearingID,
  :voluntaryIndicator,
  :participatedCode,
  :behalfOfCode,
  :comments,
  :versionNo,
  :name,
  :caseParticipantRoleID,
  :caseParticipantRoleVersionNo,
  :recordStatusCode,
  :participantRoleID 
FROM
  HEARINGWITNESS HW,
  CASEPARTICIPANTROLE CPR,
  CONCERNROLE CR 
WHERE HW.hearingWitnessID = :hearingWitnessID 
  AND HW.caseParticipantRoleID = CPR.caseParticipantRoleID 
  AND CPR.participantRoleID = CR.CONCERNROLEID
