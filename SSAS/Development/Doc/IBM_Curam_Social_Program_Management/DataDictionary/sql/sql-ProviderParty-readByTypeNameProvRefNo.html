Database SQL Operation: ProviderParty.readByTypeNameProvRefNo

Operation Details
Description: Searches the Provider Party records using name (which matches ConcernRoleName), type and reference number.
Sterotype: ns

SQL 
SELECT
  providerPartyID,
  providerConcernRoleID,
  partyConcernRoleID,
  ProviderParty.startDate,
  ProviderParty.endDate,
  category,
  ProviderParty.type,
  recordStatus
INTO
  :providerPartyID,
  :providerConcernRoleID,
  :partyConcernRoleID,
  :startDate,
  :endDate,
  :category,
  :type,
  :recordStatus 
FROM
  ProviderParty,
  ConcernRole 
WHERE ProviderParty.providerConcernRoleID = :providerConcernRoleID 
  AND ProviderParty.type = :type 
  AND ProviderParty.partyConcernRoleID = ConcernRole.concernRoleID 
  AND upper(ConcernRole.concernRoleName) =  :name 
  AND ProviderParty.category = :category 
  AND :currentDate >= ProviderParty.startDate 
  AND 
  (
    :currentDate <= ProviderParty.endDate 
    OR ProviderParty.endDate is null 
  )
