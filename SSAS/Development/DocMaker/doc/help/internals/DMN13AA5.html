Final Tasks

Your new DocBook document is now only missing one thing. The legal notice and printing history has to be added. A suitable starting point is the standard notice supplied in DocMaker's sample book templates.

  1. Locate the DocMaker installation folder and open the file templates/book/compound/book.xml.

  2. Copy the entire bookinfo element.

  3. Open the dboutput/book.xml file containing your new book and paste the bookinfo element between the book's title and the XInclude references.

  4. Update the copyright and print history and add any extra legal notices if required before saving the file.

You have now completed the conversion process. The contents of the dboutput folder can be relocated to an appropriate place in your documentation hierarchy and placed under SCM control. You should run DocMaker's spell-checking and rules-checking targets to identify any other problems that may need attention. Generating the PDF documentation and examine it; here are some of the more common issues that may need to be resolved:

  • Footnote references in the text (usually denoted by a superscript number in the original FrameMaker file) that appear inside inline mark-up may cause validation errors. This is because the footnote element is invalid inside elements such as emphasis and literal. During the conversion process, these footnotes are moved from within the element to just after it if they appear at the end of the mark-up. However, footnotes that are not at the end of the mark-up cannot be safely moved without knowledge of the context and must be moved manually.

  • Tables that have no titles can cause problems when DocMaker generates ID values for cross-references. Adding titles to the tables will fix this.

  • Words that are valid in the document but not in the spelling dictionary, for example, property names, must be added to a new word list. You can do this by running DocMaker's spell-checker.

  • Tables are inserted into FrameMaker documents using a special anchor that is usually located in the previous paragraph. When converted, the DocBook table element will be embedded in the corresponding para element and should be moved outside the paragraph.

  • Lists that contain complex content or unusual paragraph sequences may not be identified correctly and may need to be restructured.

  • Instructions describing menu selections should be updated to use the menuchoice element.

  • Quotation marks should be replaced with the quote element.
