Database Table: ScriptInformation

Table Description
IEG Script: A controlled flow through one or more Question Groups.

An Intelligent Evidence Gathering (IEG) script contains one or more groups of questions. The flow of the script is stored on an IEG Script row.

An IEG Script can control the flow of questions based on the answer(s) to previous questions.

Example

You implement new application functionality to screen claims for your unemployment Product. Your functionality requires to execute an IEG Script to gather basic evidence for the claim.

You use the IEG Editor to define a new IEG Script to gather this evidence. Your IEG Script contains the following Question Groups:

  • personal details - the claimant's name, address and date of birth;
  • cessation reason - the reason why the claimant left their last employment; and
  • last employment details - the name and address of the claimant's last employer and the date that the claimant's employment ceased.

You specify that if the reason why the claimant left their last employment was 'voluntary quit' then the script should end (since the claimant will not be eligible for your unemployment Product); otherwise the script should continue on to gather the last employment details.

The IEG Editor creates an IEG Script row to specify which Question Groups are used by the script and the logical/conditional flow between them.

Person John Smith applies for your unemployment Product and is interviewed by User Ronan Rooney. Ronan navigates to your screening application functionality which invokes the IEG Execution Infrastructure to start your script.

The IEG Execution Infrastructure reads the IEG Script row to identify which Question Groups to use, and:

  • creates a web page asking Ronan to enter John Smith's name, address and date of birth. Ronan enters this information and clicks 'Next';
  • creates a web page asking Ronan to choose from a list of reasons why John Smith left his last employment. Ronan chooses 'Lack of work' and clicks 'Next';
  • creates a web page asking Ronan to enter details of John Smith's last employment. Ronan enters these details and clicks 'Next'; and
  • displays an 'End of Script' page. Ronan clicks 'Finish'.

The script ends and control returns to your application functionality, which assesses that John may be eligible for your unemployment Product and proceeds to gather further evidence required by your Product's rules.

Person Mary Smith also applies for your unemployment Product and is interviewed by User Ronan Rooney. Ronan navigates to your screening application functionality which invokes the IEG Execution Infrastructure to start your script.

The IEG Execution Infrastructure:

  • creates a web page asking Ronan to enter Mary Smith's name, address and date of birth. Ronan enters this information and clicks 'Next';
  • creates a web page asking Ronan to choose from a list of reasons why Mary Smith left her last employment. Ronan chooses 'Voluntary quit' and clicks 'Next'; and
  • displays an 'End of Script' page. Ronan clicks 'Finish'.

The script ends and control returns to your application functionality, which assesses that Mary is not eligible for your unemployment Product.
