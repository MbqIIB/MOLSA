Database Table: Note

Table Description
Note: Free-text information relating to a participant or a case.

A user can enter free text information in the form of a case note or participant note. Once a note has been created, its content cannot be deleted or modified. However, additional information can be added to the note.

Curam records the username, date and time of each instance a note is created or added to. A sensitivity level can be set on a note to restrict access to its content. A user can only view a note if their sensitivity level is greater than or equal to the sensitivity level set on the note.

Example

James Smith is a registered participant. User Ronan Rooney creates a note for James Smith with the following content:

  • James Smith called into the office today. James advised that he is leaving the country to live in France in June 2005.

Technical Notes

The maximum amount of note text that can be entered is 30720 characters.

Many types of Curam entities can have a related note; logically these entities have a 'note' attribute which is a 'complex data type'. Physically however each of these entities has a note attribute which relates to a row on the note table.

The note entity does not perform any optimistic locking. It is the responsibility of other entities which aggregate a note to ensure that they perform any optimistic lock checking required.
