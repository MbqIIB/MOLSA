Database Table: AlternateName

Table Description
Alternate Name: A name recorded for a participant of type person.

Many names can be recorded for a person in Curam. All people must have a a primary name known as a 'primary alternate name'. A type e.g. legal name, stage name, etc. is assigned to each alternate name row.

Example

Mary Smith has two alternate name records:

  • 'Mary Smith', which is her legal name and is stored as the primary alternate name; and
  • 'Mary Jones', which was her name prior to becoming married and taking her husband's name.

Technical Notes

Logically there is a foreign key linking to the ConcernRole table; physically however it cannot be constrained via a foreign key constraint, as this would result in a circular dependency which would not allow any rows to be created on the AlternateName/ConcernRole/Person tables.
