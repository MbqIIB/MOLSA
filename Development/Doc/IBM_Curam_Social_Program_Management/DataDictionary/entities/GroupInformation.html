Database Table: GroupInformation

Table Description
Question Group: A group of questions to be answered by a User as part of an IEG Script.

An Intelligent Evidence Gathering (IEG) script contains one or more groups of questions.

Each group of questions is stored (together with its expected answer types) on a Question Group row.

When the IEG Script executes, the IEG Execution Infrastructure reads the Question Group and dynamically creates a web page containing:

  • the questions in the group; and
  • an appropriate control for the User to enter/choose an answer to each question.

Example

You implement new application functionality to screen claims for your unemployment Product. Your functionality requires to execute an IEG Script to gather basic evidence for the claim.

You use the IEG Editor to define a new IEG Script to gather this evidence. Your IEG Script contains the following Question Groups:

  • personal details - the claimant's name, address and date of birth;
  • cessation reason - the reason why the claimant left their last employment; and
  • last employment details - the name and address of the claimant's last employer and the date that the claimant's employment ceased.

You specify that if the reason why the claimant left their last employment was 'voluntary quit' then the script should end (since the claimant will not be eligible for your unemployment Product); otherwise the script should continue on to gather the last employment details.

The IEG Editor creates three Question Group rows to store your Question Groups . You can reuse any or all of these Question Groups in other scripts (e.g. perhaps all your scripts will start by gathering the claimant's name, address and date of birth).

Person John Smith applies for your unemployment Product and is interviewed by User Ronan Rooney. Ronan navigates to your screening application functionality which invokes the IEG Execution Infrastructure to start your script.

The IEG Execution Infrastructure:

  • reads a Question Group row and creates a web page asking Ronan to enter John Smith's name, address and date of birth. Ronan enters this information and clicks 'Next';
  • reads a Question Group row and creates a web page asking Ronan to choose from a list of reasons why John Smith left his last employment. Ronan chooses 'Lack of work' and clicks 'Next';
  • reads a Question Group row and creates a web page asking Ronan to enter details of John Smith's last employment. Ronan enters these details and clicks 'Next'; and
  • displays an 'End of Script' page. Ronan clicks 'Finish'.

The script ends and control returns to your application functionality, which assesses that John may be eligible for your unemployment Product and proceeds to gather further evidence required by your Product's rules.
