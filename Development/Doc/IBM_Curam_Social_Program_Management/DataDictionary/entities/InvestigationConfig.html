Database Table: InvestigationConfig

Table Description
Investigation Config: Configuration details of a type of investigation which may need to be recorded independently or during case processing.

This table stores the configuration details of types of investigations which can be created on a product delivery or integrated case or as a standalone case.

Example

During case processing a user receives information that a client may be working while at the same time claiming benefit. An investigation must be created by the user to record details of the suspicion and any actions taken to determine if the client is at fault. To facilitate this, an investigation of type Benefit Fraud must be created in the Administration component. The configuration details of the Benefit Fraud investigation are stored on the Issue Configuration table.
