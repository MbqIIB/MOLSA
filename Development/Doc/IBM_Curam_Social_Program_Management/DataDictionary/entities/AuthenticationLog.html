Database Table: AuthenticationLog

Table Description
Authentication Log: An attempt by a User to log into the application.

Each attempt to log in may be either successful or unsuccessful.

For an unsuccessful log in attempt, information is stored detailing why the attempt was unsuccessful (e.g. 'bad password').

Example

User Ronan Rooney starts the Cúram client, types in his user name and an incorrect password and attempts to log in. Cúram records the unsuccessful login attempt and Ronan Rooney's screen displays an appropriate error message.

User Ronan Rooney then re-enters his user name and his correct password and now successfully logs in to the application. Cúram records the successful login attempt.

Technical Notes

The timeEntered column stores the actual date/time of the attempted login, whereas the lastWritten column facilitates export to Cúram Reporting.

For more information see the Cúram Server Developer's Guide.
