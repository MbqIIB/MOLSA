Database Table: UpdateProcessor

Table Description
UpdateProcessor :

An Update Processor replaces or partially replaces the standard processing invoked when a data update is performed on the Data Hub. The Update Processor can be invoked prior to the standard processing, after the standard processing or instead of the standard processing. This behavior is determined by the insertPoint attribute. New behaviors can be added by implementing the CustomUpdateProcessor interface.

Example

Example 1: Jim the Developer has defined an UpdateProcessor for the "HavingABaby" Data Hub Context. It is inserted in the PRE position according to the UpdateProcessorType code table. The purpose of this UpdateProcessor to update the Citizen's Case Notes with additional information about their request.

Example 2: Joanne the developer has added an UpdateProcessor for the "ChangeOfAddress" Data Hub Context. It is inserted in the REPLACE position. This means that when the data is submitted with this Data Hub Context, it is never transformed back into Evidence update. Instead, the standard processing in replaced with new Custom Processing which sends the update to a legacy system in the form of a web service call.
