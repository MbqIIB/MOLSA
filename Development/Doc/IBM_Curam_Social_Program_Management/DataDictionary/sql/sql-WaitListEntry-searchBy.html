Database SQL Operation: WaitListEntry.searchBy

Operation Details
Description: Searches the wait list entries for a provider for the search criteria entered. Search criterion can be Client Name, Priority, Expiry Date and/or Status. Search is performed based on partial match of the client name criteria.
Sterotype: nsmulti

SQL 
SELECT
  WaitListEntry.waitListEntryID,
  WaitListEntry.waitListID,
  WaitListEntry.caseParticipantRoleID,
  WaitListEntry.position,
  WaitListEntry.priority,
  WaitListEntry.expiryDate,
  WaitListEntry.removalReason,
  WaitListEntry.status,
  WaitListEntry.versionNo,
  WaitListEntry.reviewDate,
  WaitListEntry.comments
INTO
  :waitListEntryID,
  :waitListID,
  :caseParticipantRoleID,
  :position,
  :priority,
  :expiryDate,
  :removalReason,
  :status,
  :versionNo,
  :reviewDate,
  :comments 
FROM
  ConcernRole,
  WaitListEntry,
  WaitList,
  CaseParticipantRole 
WHERE 
  (
    :searchByName = '0' 
    OR upper(ConcernRole.concernRoleName) like :name 
  ) 
  AND 
  (
    :searchByPriority = '0' 
    OR WaitListEntry.priority = :priority 
  ) 
  AND 
  (
    :searchByExpiryDate = '0' 
    OR WaitListEntry.expiryDate = :expiryDate 
  ) 
  AND 
  (
    :searchByStatus = '0' 
    OR WaitListEntry.status = :status 
  ) 
  AND WaitList.waitListID = WaitListEntry.waitListID 
  AND WaitList.resourceID = :resourceID 
  AND WaitList.type = :waitListType 
  AND WaitListEntry.caseParticipantRoleID = CaseParticipantRole.caseParticipantRoleID 
  AND CaseParticipantRole.participantRoleID = ConcernRole.concernRoleID 
ORDER BY WaitListEntry.position ASC
