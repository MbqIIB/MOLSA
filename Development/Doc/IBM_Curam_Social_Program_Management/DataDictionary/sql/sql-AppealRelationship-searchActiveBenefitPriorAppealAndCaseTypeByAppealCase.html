Database SQL Operation: AppealRelationship.searchActiveBenefitPriorAppealAndCaseTypeByAppealCase

Operation Details
Description: Returns the priorAppealCaseID, caseID, caseType for the caseID, continueBenefitsIndicator, appealRelationshipID and versionNo for all active appealed cases for an appeal.
Sterotype: nsmulti

SQL 
SELECT
  AppealRelationship.appealRelationshipID,
  AppealRelationship.versionNo,
  AppealRelationship.continueBenefitsIndicator,
  AppealRelationship.priorAppealCaseID,
  AppealRelationship.caseID,
  AppealRelationship.statusCode,
  caseTypeCode
INTO
  :appealRelationshipID,
  :versionNo,
  :continueBenefitsIndicator,
  :priorAppealCaseID,
  :caseID,
  :statusCode,
  :caseTypeCode 
FROM
  AppealRelationship,
  CaseHeader 
WHERE AppealRelationship.caseID = CaseHeader.caseID 
  AND AppealRelationship.appealCaseID = :appealCaseID 
  AND AppealRelationship.recordStatus = :recordStatus
