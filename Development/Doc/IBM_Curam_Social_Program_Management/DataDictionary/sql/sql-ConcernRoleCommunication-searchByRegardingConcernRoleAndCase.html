Database SQL Operation: ConcernRoleCommunication.searchByRegardingConcernRoleAndCase

Operation Details
Description: This method will return a list of communications and a case reference value based on the regarding ConcernRole and the Case.
Sterotype: nsmulti

SQL 
SELECT
  ConcernRoleCommunication.communicationID,
  ConcernRoleCommunication.correspondentName,
  ConcernRoleCommunication.correspondentConcernRoleID,
  ConcernRoleCommunication.subjectText,
  ConcernRoleCommunication.typeCode,
  ConcernRoleCommunication.communicationDate,
  ConcernRoleCommunication.communicationStatus,
  ConcernRoleCommunication.statusCode,
  ConcernRoleCommunication.ticketID,
  ConcernRoleCommunication.versionNo,
  ConcernRoleCommunication.methodTypeCode,
  ConcernRoleCommunication.attachmentInd,
  ConcernRoleCommunication.caseID,
  ConcernRoleCommunication.communicationFormat,
  CaseHeader.caseReference
INTO
  :communicationID,
  :correspondentName,
  :concernRoleID,
  :subjectText,
  :typeCode,
  :communicationDate,
  :communicationStatus,
  :statusCode,
  :ticketID,
  :versionNo,
  :methodTypeCode,
  :attachmentInd,
  :caseID,
  :communicationFormat,
  :caseReference 
FROM
  ConcernRoleCommunication,
  CaseHeader 
WHERE
(
ConcernRoleCommunication.concernRoleID =  :concernRoleID
OR 
   (
    ConcernRoleCommunication.concernRoleID IS NULL AND     
    ConcernRoleCommunication.correspondentConcernRoleID  = :concernRoleID
   )
 )
  AND CaseHeader.caseID = :caseID 
  AND ConcernRoleCommunication.caseID = CaseHeader.caseID 
ORDER BY ConcernRoleCommunication.communicationStatus,
  ConcernRoleCommunication.communicationDate DESC
