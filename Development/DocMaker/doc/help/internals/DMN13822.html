Creating the book.xml File

FrameMaker produces no useful conversion of the main book file, so the DocBook XML equivalent must now be created. This file, book.xml, contains references to all the other files in the book and allows DocMaker to process the book as a whole. DocMaker can create this file automatically, but you will still need to check it manually and possibly correct the ordering of the content.

  1. Open a command prompt, and change the current folder to the folder containing your book files and the xmloutput folder.

  2. At the command prompt, run the following command:

    C:\MyBook> docmaker prepare-fm

    This will create a book.xml file in the xmloutput folder. The book will contain a reference to every XML file (in alphabetical order) created by FrameMaker. Files that match the name patterns *TOC.xml, *PrintHistory.xml, *Cover.xml, *IX.xml (but not *Unix.xml), and *.backup.xml will be excluded automatically. If you have a file containing content that matches one of these patterns, you must add it to the book.xml file manually.

  3. Open the xmloutput/book.xml file in an editor and locate the line defining the title of the book: 

    <book>
  <title>Unknown</title>
  ...
</book>

    Change the text within the title element to the correct title of your book.

  4. Below the book title, you will see XInclude references for each main content file in your book. Reorder the references if necessary. If a file was not included (perhaps one of the file names has the same ending as the files that are automatically excluded), add a new reference in the appropriate place within the book element copying the format of those present.

Once the book.xml file is ready, you can now convert your documents to correct DocBook XML format. If you encounter errors, you will have to fix them in the original files, but you will not have to recreate the book.xml file.
