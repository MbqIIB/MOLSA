Processing Graphics

While FrameMaker exports all the diagrams and images in your document to individual files, it does not embed adequate information in the XML documents for DocMaker to create proper references to the graphic files. You will need to rename the graphic files and check the references to these files created by DocMaker. FrameMaker was also instructed to output graphic files only in the GIF (Graphics Interchange Format) and WMF (Windows Mediafile) formats. The former is used for all images and the latter for diagrams drawn within FrameMaker. The WMF files must be converted to SVG format to be compatible with DocMaker.

Converting WMF to SVG

The OpenOffice.org Draw application can do this type of conversion. Open the WMF file, select the graphic, and then use the File→Export... menu option to export the file to SVG format.

SVG Containing Bitmaps

If a WMF format image file contains both a bitmap (e.g., a screen-shot) and drawing elements (e.g., arrows and text), then DocMaker may not be able to process the SVG file created during conversion. It is best to convert these files to GIF image files.

The names of the image files match those of the original FrameMaker file in which they appeared, though with the addition of a “sequence number” and a different extension. For example, the fifth image that was in the file UserGuide.fm would be output to a file named UserGuide-5.gif. The sequence numbers begin anew with each file in the book; books that contained only one main content file will result in image files with unique sequence numbers; books that contained many files will result in image files with sequence numbers that repeat as each file is processed. DocMaker can preserve the sequence numbers, but cannot determine the proper file names, so these must be fixed by hand.

DocMaker must invent file names for its references to the image files because the original file names are unavailable. The names that DocMaker uses are composed from the chapter or appendix name and the image sequence number. If the book contained a single main content file, it is just a matter of finding the imagedata elements in the DocBook XML files and renaming the image file with the same sequence number to match the value of the fileref attribute. For example, the value of the fileref attribute might be images/concepts-34.gif (meaning the “Concepts” chapter and the thirty-fourth image that appeared in the file containing that chapter), so you should locate the image file with sequence number 34 and rename it to concepts-34.gif. If the book contained more than one main content file, you may have more than one image file with the same sequence number and you must ensure both that the sequence numbers match and that the original FrameMaker file, whose name is used in the name of the image file, contained the chapter whose name is used in the fileref attribute reference.

This process is relatively simple, but there is a possible cause of confusion. If a chapter was split over several files with sections in these file containing images, then the reference in the DocBook XML file may be ambiguous. In this case, the reference will use the chapter name and sequence number, but as there were several files in the chapter, the sequence number in the image files will have been repeated. For example, images/development-2.gif could refer to the second image in any of the original files that contributed content to the “Development” chapter; you may see this reference repeated more than once if more than one file contained two or more images. To resolve this, you will need to identify the section that contains the reference and then match that to the image file named after the file that contained that section; the sequence numbers will then match as before. However, you will need to assign new numbers and change the value of the fileref attributes so that all these references are uniquely named.

Note

When renaming files, there is no need to keep the sequence numbers; these are just a convention used in the conversion process. You can rename the files to anything you like (you could base the name on the title of the figure), as long as you change the value of the fileref attribute to match. You can also organize the image file in sub-folders of the images folder and, again, update the reference in the fileref attribute.

The following steps must be followed for each file in the dboutput/images folder.

  1. If the file extension is .wmf convert the file to SVG format with an extension of .svg (see above).

    You can keep the WMF format file if you wish to use it as the master copy of the graphic, but if you change it, you must update the SVG file.

  2. View the image file and then locate the matching figure in your original FrameMaker document. The exported graphic files have the same name as the original document file and an additional sequence number. This should make it easier to locate. Once located, note the figure title.

  3. Open the DocBook XML file in dboutput for the chapter or appendix that contains the figure and locate the figure element that contains the reference to the graphic file. The file name will be different, so use the figure title that you noted in the previous step to identify the correct figure element.

    For figures without titles, you will have to locate the corresponding informalfigure element. As there is no title, use the sequence number and the context as a guide.

  4. Within the figure element you will see an imagedata element containing a reference to an image file. Note the value of this fileref attribute. If the extension on the file name is .gif, but the graphic file is in SVG format, change the extension to .svg and change the value of the format attribute to SVG.

  5. Rename the graphic file to match the name in the fileref attribute. Note, however, that if the original document file contained only sections and no chapter or appendix title, that you may need to assign a new sequence number as described above.
