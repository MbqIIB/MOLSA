Installation

DocMaker is released as a simple ZIP file that expands to a DocMaker folder containing all of the necessary files. When replacing an existing installation of DocMaker, the old installation should be deleted before expanding the new ZIP file. The installation steps are as follows:

  1. Ensure that you have a Java Runtime Environment (JRE) or Java Development Kit (JDK) with version 1.3 or higher installed.

  2. Obtain the DocMaker installation ZIP file.

  3. Extract the ZIP file to your hard disk. The DocMaker files will automatically be placed in a DocMaker folder.

  4. Set the DOCMAKER_HOME environment variable to point to your DocMaker installation folder. For example, if you installed DocMaker in the C:\DocMaker folder, you should set the DOCMAKER_HOME environment variable to C:\DocMaker.

    Important

    When executing commands at a command prompt, it is usual to place quotation marks around the names of files and folders that contain space characters. However, when setting the DOCMAKER_HOME environment variable, you must not include quotation marks around the name of the installation folder.

  5. Add DocMaker to your system path by editing the PATH system environment variable and adding %DOCMAKER_HOME%\bin to it.

The contents of a DocMaker release (with paths relative to the DocMaker folder) are shown in the table below. Not all of the files that are released are necessary for all operations and some should not be released externally, as they are not supported for use by customers.

File or Folder

Description

bin/

Contains the Ant scripts and Windows batch scripts for executing DocMaker.

bin/docmaker.bat

The Windows shell script that executes DocMaker directly from the command-line. Where DocMaker is invoked directly from other scripts, this file is may not be necessary.

bin/docmaker.xml

The main DocMaker Ant script that defines all of the supported DocMaker targets.

bin/dm-internal.xml

The DocMaker Ant script that defines the internal, unsupported DocMaker targets, including plain HTML generation and FrameMaker file conversion. This should not be shipped to customers.

bin/docmaker.properties

An Ant properties file that defines some values that are used by the DocMaker Ant script.

bin/wrapexit.bat

A special Windows shell script that can be used to correctly detect errors if the docmaker.bat script is invoked from an Ant script.

doc/

The DocMaker documentation in PDF and HTML Help format.

doc/format/internal/

The Cúram DocMaker Internals manual covering all aspects of DocMaker use including features only supported internally.

doc/format/manual/

The Cúram DocMaker Manual, an abridged version of the DocMaker manual for distribution to customers as part of the Documentation Center development environment.

doc/help/images/

The boilerplate images and icons that must be redistributed with any HTML Help version of the DocMaker manuals.

lib/

All of the stylesheets and tools used by DocMaker. The files in this folder include the main stylesheets for pre-processing the DocBook content before rendering and the Java libraries for DocMaker Ant tasks and the HTML Help Applet. Files matching the pattern fm*.* should not be shipped to customers, as they are for FrameMaker file conversion only.

lib/docmaker.jar

The tools used by DocMaker when rendering documents. Some of the classes in this JAR file are also found in help-applet.jar, as they are also needed by the Help Applet.

lib/help-applet.jar

The Help Applet classes and the tools necessary to support the operation of the applet. To keep this JAR file small, a small subset of the tools found in docmaker.jar are included, but that JAR file is not required by the applet.

lib/help-server.jar

The Help Server classes used to support a server-side alternative to the Help Applet. Management of the table of contents, searching and context sensitive look-ups are performed by Java Servlets deployed in a J2EE web container rather than by the Help Applet. The Help Server shares much of its implementation with the Help Applet, so the applet JAR file must be available on the server.

lib/jde-commons-ext.jar

A special build of the JDE Commons library that uses the Java 1.5 compiler to ensure that all timing facilities are included to enable accurate profiling (see DocMaker Tools for more details). This need not be shipped to customers.

lib/profile.jar

Special profiling classes used internally for performance metrics, etc. (see DocMaker Tools for more details). This need not be shipped to customers.

lib/profile-ant.jar

Special profiling classes used internally for performance metrics, etc. (see DocMaker Tools for more details). This need not be shipped to customers.

lib/apache-ant/

The Apache Ant Java-based build tool. DocMaker includes this for the benefit of users who do not have a typical Cúram development environment available.

lib/ant-contrib/

Additional Ant tasks from the Ant-Contrib project.

lib/docbook-xml/

The DocBook XML 4.2 DTD from OASIS used when validating content to avoid the need to download the DTD dynamically using the URL at the top of every DocBook XML file.

lib/docbook-xsl/

The DocBook XSL stylesheet distribution from the Open DocBook Repository. Most of the rendering to HTML Help and XSLFO format are performed by these stylesheets.

lib/dtd-custom/

A custom extension to the DocBook XML 4.2 DTD that permits the use of xi:include elements in DocBook content.

lib/fonts/

Font description file for the Trebuchet MS and Arial Narrow fonts that Apache FOP requires for PDF rendering.

lib/fop/

The Apache FOP library that converts XSLFO documents to PDF documents.

lib/images/

The boilerplate icons, logos, and other images used when rendering documents.

lib/lucene/

The Apache Lucene library that performs the indexing and searching functions required by the Help Applet.

lib/xalan-j/

The Apache Xalan XSLT processor that performs XSLT stylesheet transformations.

lib/xsl-custom/

Contains the stylesheets developed to customize the Open DocBook Repository's stylesheets to produce output suitable for Cúram.

lib/xsl-custom/common/

Configuration setting, localized strings, and performance tweaks applied to the third-party stylesheets and used for all output formats.

lib/xsl-custom/fo/

Customizations for XSLFO output used in PDF generation.

lib/xsl-custom/help/

Customizations for HTML Help output used for online help.

lib/xsl-custom/html/

Unsupported customizations for plain HTML output. This need not be shipped externally.

templates/

Simple DocBook XML files that can be used as templates when creating new content.

README.txt

A text file indicating the location of the DocMaker manual within the installation. There is no need to ship this externally.

VERSION.txt

A text file containing only the version number of this release of DocMaker. This can be useful when build scripts need to test if the installed version number matches the expected version number.
