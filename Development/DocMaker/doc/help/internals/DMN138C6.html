Converting to DocBook XML

FrameMaker cannot convert your documents into correct DocBook XML, though it does come close, so DocMaker will now take over and complete the conversion process. DocMaker will also restructure your document and place one chapter or appendix in each output file. The files are named using the normalized title (see the section, Cross-References for details) of the chapter or appendix and a prefix. For chapters, the prefix is “c_” and for appendices, “a_”. Any graphic files will be copied and placed in an images sub-folder of your document folder.

  1. Using your file manager, go to the folder containing your book files and create a new sub-folder named dboutput. This will be used to hold the final DocBook XML files and the images sub-folder containing all the graphic files.

  2. Open a command prompt, and change the current folder to the folder containing the dboutput, and xmloutput folders.

  3. At the command prompt, run the following command:

    C:\MyBook> docmaker convert-fm

    DocMaker will now convert the xmloutput/book.xml file to DocBook XML format and, as this file includes all the others, all of your documentation will be processed. Each chapter and appendix will be placed in a separate file and a new book.xml file will be created that references them. Graphic files (identified by DocMaker as those files that do not have a *.xml extension) will be copied to the dboutput/images folder.

    The chapter and appendix files are named using the title of the chapter or appendix. Some titles containing characters such as “ú” may cause problems during builds. You can fix these by renaming the file and then renaming the references to the file in book.xml.

    If your document contains a code sample that uses an XML CDATA block, it is likely that FrameMaker will not generate valid XML. If you get an error caused by this, edit the XML file and change the block's closing “]]>” sequence to “]]&gt;”.

    If an error message appears reporting that a cross-reference could not be resolved, it is nearly always caused by the use of the wrong cross-reference format. For example, a cross-reference to a figure that is formatted using the “Table Reference” format will cause such a problem.

The conversion process is now complete. The new files in dboutput are in valid DocBook XML format. However, a few clean-up tasks remain.
