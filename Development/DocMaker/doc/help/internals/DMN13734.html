Converting to XML

The next task is to convert the structured FrameMaker documents to XML format. A structure application document is used to control FrameMaker when it is writing the XML output and converting embedded graphics.

  1. Using your file manager, go to the folder containing your original book files and create a new sub-folder named xmloutput. This will be used to hold the saved XML files and graphic files that will later be converted to DocBook format.

  2. Go to the fmoutput folder containing your newly structured document files and remove any backup files created during the structuring process.

  3. In FrameMaker, open the file named fm2fmdb_app.fm located in the lib folder of your DocMaker installation.

  4. Change the value of the DOCMAKER_HOME variable to the path to your DocMaker installation folder by selecting Special→Variable... from the menu. In the dialog box, select the DOCMAKER_HOME variable from the Variables list. Press the Edit Definition... button. In the next dialog box, change the path in the Definition field to the path to your DocMaker installation. Note, you must type two back-slash characters for each back-slash used in the path, for example, use C:\\DocMaker and not C:\DocMaker. Press the Done button in each dialog box to apply the new variable value and save the file. You will only need to do this once unless the location of your DocMaker installation changes or it is reinstalled.

  5. From the main menu, select File→Structure Tools→Read Application Definitions. You will not need to do this again unless you restart FrameMaker.

  6. From the main menu, select File→Utilities→Convert Documents to Structured Format....

  7. In the dialog box, set the Application to “CúramDocMaker”. In the Input FrameMaker Files section, set the Directory field to the location of this book's fmoutput folder and set the Suffix field to “fm”. In the Output Structured Documents section, set the Directory field to the location of the xmloutput folder you created in the first step and set the Suffix field to “xml”. Check the Allow Existing Files to Be Overwritten check-box and press the Convert button. Check the log window for any errors and correct them. Errors reported against files that do not contain chapters or appendices can be ignored.

You should now see XML files in your xmloutput folder. These are in an intermediate format that will soon be converted to correct DocBook XML.
