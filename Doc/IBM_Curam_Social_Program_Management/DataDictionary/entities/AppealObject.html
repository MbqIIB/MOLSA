Database Table: AppealObject

Table Description
Represents the case objects associated with an appeal relationship record under appeal. Appeal relationship records may have many associated appeal objects. An appeal object can represent any item which has a related caseID. For example, a program application which has been denied is an object associated with an Application. An Application contains a caseID as it is modelled on an integrated case type. Each object has a ?type? entry. For example, if a denied program application is being appealed, an entry will be added to the appeal object of type ?Program Application?. The implementation of appeals in the application allows for a one to one relationship between an AppealRelationship and AppealObject record. This is to allow Users to decide on each object under appeal separately. For example, an application for Income Support contains two programs ? Cash Assistance and Food Assistance. Both programs are denied. The customer submits an appeal because they disagree with the grounds for denial. Two AppealRelationship records are created under the Appeal ? one linked to the Cash Assistance program application AppealObject and one linked to the Food Assistance AppealObject. The evidence presented at the appeal hearing results in the Food Assistance program denial under appeal being accepted in the clients favour and the Cash Assistance program denial under appeal being rejected.
