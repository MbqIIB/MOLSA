Database SQL Operation: InstructionLineItem.searchILIByFinInstructID

Operation Details
Description: To get a list of Instruction Line Items and case reference that were combined into one Financial Instruction. The caseReference returned will be null in situations where the ILI does not relate to a case, using an outer join to allow ILIs with a null caseID to be retrieved.
Sterotype: nsmulti

SQL 
SELECT
  ili.instructLineItemID,
  ili.amount,
  ili.unprocessedAmount,
  ili.instructLineItemCategory,
  ili.instructionLineItemType,
  ili.financialCompID,
  ili.coverPeriodFrom,
  ili.coverPeriodTo,
  ili.effectiveDate,
  ili.creationDate,
  ili.statusCode,
  ili.caseID,
  ili.finInstructionID,
  ili.concernRoleID,
  ili.caseNomineeID,
  ili.primaryClientID,
  ili.deliveryMethodType,
  ili.creditDebitType,
  ili.currencyExchangeID,
  ili.currencyTypeCode,
  ili.adjustmentInd,
  ili.adjustmentFrequency,
  ili.nextAdjustmentDate,
  ili.instrumentGenInd,
  ili.fundID,
  ili.maximumAmount,
  ch.caseReference,
  ili.versionNo
INTO
  :instructLineItemID,
  :amount,
  :unprocessedAmount,
  :instructLineItemCategory,
  :instructionLineItemType,
  :financialCompID,
  :coverPeriodFrom,
  :coverPeriodTo,
  :effectiveDate,
  :creationDate,
  :statusCode,
  :caseID,
  :finInstructionID,
  :concernRoleID,
  :caseNomineeID,
  :primaryClientID,
  :deliveryMethodType,
  :creditDebitType,
  :currencyExchangeID,
  :currencyTypeCode,
  :adjustmentInd,
  :adjustmentFrequency,
  :nextAdjustmentDate,
  :instrumentGenInd,
  :fundID,
  :maximumAmount,
  :caseReference,
  :versionNo 
FROM
  InstructionLineItem ili 
  
  LEFT OUTER JOIN
  
  CaseHeader ch 
  ON ch.caseID = ili.caseID 
WHERE ili.finInstructionID = :finInstructionID 
ORDER BY ili.finInstructionID
